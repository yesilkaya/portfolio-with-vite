🌐 TCP ve HTTP Nedir? Neden İkisi de Gerekli?
🔹 1. TCP (Transmission Control Protocol)
🧱 Görevi:
Verilerin eksiksiz, sıralı ve güvenli iletilmesini sağlar.

Bağlantı temellidir (connection-oriented): Veri göndermeden önce bağlantı kurar.

🔄 Bağlantı Kurma: 3 Aşamalı El Sıkışma (3-Way Handshake)
SYN → Tarayıcı sunucuya bağlantı isteği gönderir.

SYN-ACK → Sunucu isteği kabul eder ve onay gönderir.

ACK → Tarayıcı onayı alır, bağlantı kurulur.

📦 Veri Aktarımı:
Veri paketlere bölünerek gönderilir.

Her paketin numarası ve kontrol toplamı (checksum) vardır.

Alıcı eksik ya da bozuk paketleri bildirir → TCP tekrar gönderir.

Paketler sıraya dizilir: Örn. 3,1,2 → TCP: 1,2,3 yapar.

🔚 Bağlantı Kapatma: 4 Aşamalı Süreç
FIN → İstemci bağlantıyı kapatmak ister.

ACK → Sunucu kabul eder.

FIN → Sunucu da kapatmak ister.

ACK → İstemci onaylar.

✅ TCP'nin Özellikleri:
Özellik	            Açıklama
Güvenilir	        Veri eksiksiz ve doğru sırada iletilir.
Bağlantı Temelli	Veri göndermeden önce bağlantı kurulur.
Akış Kontrolü	    Yavaş alıcılara göre hız ayarlanır.
Hata Kontrolü	    Paket kaybı veya bozulma durumunda yeniden gönderilir.

🔹 2. HTTP (Hypertext Transfer Protocol)
🧱 Görevi:
Web sunucuları ve tarayıcılar arasında anlamlı veri (HTML, JSON, vs.) transferini sağlar.

Stateless: Her istek bağımsızdır, önceki istek hatırlanmaz.

🔌 Nasıl Çalışır?
HTTP, TCP üzerinden çalışır.
(HTTP → TCP → IP → Fiziksel Ağ)

Örnek istek:

GET /anasayfa HTTP/1.1
Host: www.example.com
🧠 HTTP’nin Sağladığı:
Ne gönderildiğini belirtir (GET, POST, HTML, JSON...).

Tarayıcı → “/anasayfa verisini istiyorum”
TCP → “Tamam, sıralı ve güvenli şekilde taşıyacağım”

🔄 TCP ve HTTP Farkları
Özellik	            TCP	HTTP
Katman	            Taşıma (Layer 4)	Uygulama (Layer 7)
Amaç	            Veriyi sıralı, güvenli taşımak	Anlamlı veri transferi yapmak
Bağlantı	        Bağlantı temelli	Bağlantısız (stateless)
Veri Formatı	    Ham veri (binary)	Metin tabanlı (HTML, JSON vs.)
Kullanım Alanı	    VPN, SSH, e-posta, web	Web tarayıcıları, API’ler

📬 Gerçek Hayat Benzetmesi
Rol	Ne Yapar?
📦 HTTP	Gönderilen içeriği açıklar ("Mesaj")
🚚 TCP	İletiyi güvenli ve sıralı taşır

➕ HTTPS = HTTP + TLS → Şifrelenmiş, güvenli iletişim

💡 Neden HTTP, IP Üzerinden Gitmez?
Çünkü:

IP güvenli değildir → Paketler eksik ya da karışık gelebilir.

TCP, paketleri sıraya dizer, eksikleri tamamlar, hataları düzeltir.

HTTP gibi anlamlı protokoller için güvenli zemin sunar.

🆕 Not: HTTP/3 Farklıdır
HTTP/3, TCP yerine QUIC (UDP tabanlı) kullanır.

Amaç: Daha hızlı bağlantı ve daha iyi güvenlik.

✅ Özetle
TCP → "Nasıl gönderileceğini" çözer (güvenli, sıralı, hatasız)

HTTP → "Ne gönderildiğini" tanımlar (GET, POST, JSON...)

Birlikte: Anlamlı ve güvenli internet iletişimi sağlar.
